# Agentic Harness

**定义**: 围绕模型搭建的约束、反馈闭环与上下文工程系统  
**类比**: Model = CPU, Context Window = RAM, Agent Harness = OS, Agent = Application  
**核心作用**: 释放模型能力的关键变量

---

## 核心悖论

**两个矛盾观点同时成立**:
1. **Agent不是一个赛道**（没有持久壁垒）
2. **Harness是效果的决定性变量**

**解释**: Harness的优势有保质期，不同层面保质期不同。

---

## 保质期结构

| 层面 | 保质期 | 特点 |
|------|--------|------|
| **短期优势** | 下一代模型发布时归零 | 模型进步抵消 |
| **长期优势** | 随模型进步增值 | 持续价值 |

**关键**: 识别哪些优势会归零，哪些会增值。

---

## Harness vs Agent

| 维度 | Harness | Agent |
|------|---------|-------|
| **定义** | 约束、反馈闭环、上下文工程 | 具体应用 |
| **类比** | 操作系统（OS） | 应用程序（App） |
| **护城河** | 有保质期 | 几乎没有 |
| **价值** | 当下效果差异的核心 | 容易被复制 |

---

## 关键组件

### 1. 约束系统
- 定义Agent能做什么、不能做什么
- 安全边界控制

### 2. 反馈闭环
- Agent执行结果的反馈
- 持续优化行为

### 3. 上下文工程
- 管理和优化上下文窗口
- 长期记忆维护

---

## 战略判断

**决策框架**:
1. **识别保质期**: 哪些优势短期归零，哪些长期增值
2. **押注窗口**: 在窗口关闭前，押注在长期增值的层面
3. **持续迭代**: 随着模型进步，调整Harness策略
