# 光学压缩（Contexts Optical Compression）

**类型**: AI技术/上下文优化  
**领域**: 大语言模型、多模态、长上下文处理  
**提出者**: DeepSeek 团队（魏浩然、孙耀峰、李雨坤）  
**首次发表**: 2025 年 10 月  
**论文**: [arXiv:2510.18234](https://arxiv.org/abs/2510.18234)  
**开源代码**: [GitHub](https://github.com/deepseek-ai/DeepSeek-OCR)

---

## 核心思想

**把文档当图片处理，用视觉 token 代替文本 token，实现 10-20 倍的上下文压缩。**

传统 LLM 处理长文本时，每个字/词都是一个 token，注意力计算复杂度随 token 数量平方增长。光学压缩的思路是：

> 一张图片承载的信息密度远高于等效文本——因为**二维排版天然携带语义**（位置、字号、框线等都是信息），不需要逐字翻译。

### 直观类比

描述一张火车票：
- **文本方式**："G123次，北京南→上海虹桥，14车08A号，票价553元"——30-40 个 token
- **图片方式**：拍一张照片——**4 个视觉 token**就够（方框=座位，大字=金额，位置=含义）

**照片的信息量远大于文字描述，但传递速度反而更快。**

## 技术架构：两段式设计

```
图片 → DeepEncoder（编码+压缩） → 视觉token（大幅减少） → DeepSeek 3B MoE（解码还原文字）
```

### DeepEncoder（核心引擎）

由三个组件串联：

| 组件 | 作用 | 参数量 |
|------|------|--------|
| SAM（窗口注意力） | 高分辨率图像的局部感知 | 80M |
| 16倍卷积压缩器 | token 数量砍掉 16 倍 | 极小 |
| CLIP（全局注意力） | 提取全局视觉知识 | 300M |

**压缩流程**（以 1024×1024 图片为例）：
1. 图片切分为 **4096 个 patch token**
2. SAM 做局部感知（窗口注意力，内存可控）
3. 卷积压缩器：4096 → **256 token**（16 倍压缩）
4. CLIP 接手全局理解（token 已很少，不爆显存）

### 解码器：DeepSeek 3B MoE

- 激活参数仅 **5.7 亿**
- 将 256 个视觉 token "翻译"回文字
- MoE 架构保证推理效率（小模型的效率，大模型的能力）

## 压缩效果

| 文档字数 | 视觉token数 | 压缩比 | OCR精度 |
|---------|------------|--------|---------|
| 600-700 字 | 100 | 6.7× | 98.5% |
| 800-900 字 | 100 | 8.5× | 96.8% |
| 900-1000 字 | 100 | 9.7× | 96.8% |
| 1000-1100 字 | 100 | 10.6× | 91.5% |
| 1200-1300 字 | 64 | 19.7× | 59.1% |

**关键阈值**：压缩比 10× 以内，精度 >96%。20× 压缩仍有约 60% 精度。

## 算力节省原理

LLM 注意力计算复杂度 = O(n²)，其中 n 是 token 数量。

| 方式 | token数 | 注意力计算量 |
|------|---------|-------------|
| 纯文本 | 1000 | 1000×1000 = **1,000,000** |
| 光学压缩 | 100 | 100×100 = **10,000** |

**理论节省 100 倍**（实际约 10-20 倍，因还有其他开销）。

### 实测性能
- 单张 A100-40G：每天 **3300 万页**
- OmniDocBench：用 <800 token/页 击败 6000+ token/页 的 MinerU 2.0

## 战略意义

### 1. 长上下文问题的另类解法
不改进注意力机制，而是**改变输入形态**——把文本变成图，token 数量暴降。

### 2. V4 百万上下文的技术底座
V4 如果直接用文本 token 处理百万上下文，算力成本是天文数字。用光学压缩：
- 100 万文本 token → 约 **5-10 万视觉 token**
- 这就在可控范围内了

### 3. LLM 记忆压缩的启发
论文自述："对历史长上下文压缩和记忆遗忘机制的研究有重要意义。"

这意味着未来 LLM 可能不需要"记住所有文字"，而是"记住视觉快照"——就像人类回忆场景时，先浮现画面，再逐字回想。
