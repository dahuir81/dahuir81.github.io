# 框架级安全 vs 模型级安全

**类型**: 概念（AI 系统架构）
**关联文章**: [[2026-04-09-Hermes-Agent-vs-OpenClaw-source]]
**创建/更新**: 2026-04-09

## 定义

两种不同的 AI Agent 安全保障思路：

### 模型级安全
依赖大模型自身的判断力来识别和阻止危险操作。

### 框架级安全
在 AI 框架层面强制执行安全策略，不依赖模型的"聪明程度"。

## 对比

| 维度 | 模型级安全 | 框架级安全 |
|------|-----------|-----------|
| 安全主体 | 大模型 | 框架代码 |
| 可靠性 | 取决于模型能力，会"犯浑" | 确定性规则，稳定 |
| 漏洞案例 | 邮件被清空、CVE-2026-25253 | 无（理论上） |
| 性能开销 | 低 | 中（额外扫描） |

## 框架级安全的具体措施

以 [[Hermes Agent]] 为例：
1. **命令执行授权审批**: 敏感操作弹出确认界面（单次/本次会话/永久允许/拒绝）
2. **Skills 安装安全扫描**: 自动检查数据泄露、提示词注入，危险直接拒绝
3. **容器隔离**: 限制操作范围
4. **凭证泄露拦截**: 防止 API Key 等敏感信息泄露
5. **受保护目录禁止访问**: 核心数据区域隔离
6. **路径穿越防护**: 防止 `../` 类攻击
7. **记忆注入前安全扫描**: 防止提示词注入攻击

## 意义

框架级安全代表了 AI Agent 安全性的正确方向——不能把安全寄托在模型"够聪明"上。即使使用能力较弱的模型，框架级安全也能保证底线。
