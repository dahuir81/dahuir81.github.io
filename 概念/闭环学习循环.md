# 闭环学习循环

**类型**: 架构模式 / 方法论
**来源**: [[Hermes Agent]] 核心设计

## 定义
Agent 在执行任务后，通过后台 review 流程自动将经验沉淀为可复用 Skill 的机制。

## 工作流程
```
前台执行任务
    ↓
累计 N 轮工具迭代（默认 10）
    ↓
响应结束后触发后台 review
    ↓
子 Agent 分析对话历史
    ↓
判断是否值得沉淀为 Skill
    ↓
保存/更新 Skill 文件（或"Nothing to save"）
```

## 关键设计
- **best-effort**: 不是硬编码必达动作，子 Agent 可判断"不值得保存"
- **后台异步**: 不阻塞用户下一轮对话
- **系统提示引导**: SKILLS_GUIDANCE 建议 5+ 工具调用时保存 Skill
- **自我迭代**: Skill 发现问题时立即 patch，不等待

## 与 OpenClaw 对比
| 维度 | OpenClaw | Hermes |
|------|----------|--------|
| Skill 来源 | 人工编写 + 社区贡献 | Agent 自动沉淀 |
| 更新机制 | 手动更新 | 后台 review + 自动 patch |
| 触发方式 | 安装时加载 | 动态扫描 + 按需加载 |
