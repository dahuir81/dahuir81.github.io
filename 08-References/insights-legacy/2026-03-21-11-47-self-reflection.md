### 1. ECS配置事故：从"执行模式"到"理解模式"的认知升级

**时间**: 2026-03-21 00:19
**严重程度**: 🔴 高

**事件经过**:
1. ECS服务器重置为OpenClaw镜像后需要配置
2. Tars未了解镜像启动机制，直接复制本地Mac配置
3. 配置验证错误 → 端口8080被占用 → 多次尝试停止进程/重启服务器
4. 最终发现是systemd用户服务自动启动导致

**慧哥的关键反馈**:
> "看来你还是不来了解，这个ECS的运行规则，你是不是应该好好学习一下相关文档"

**认知升级**:
```
错误顺序: 执行 → 遇到问题 → 猜测 → 再执行
正确顺序: 理解 → 诊断 → 执行 → 验证
```

**核心教训**:
- **新环境 ≠ 旧环境**: 阿里云镜像有预设的systemd服务，不能照搬本地配置
- **诊断先于治疗**: 端口冲突时，应先`systemctl status` + `lsof -i :8080`确认占用者
- **文档是第一道防线**: 操作前必须阅读镜像文档，了解启动机制
