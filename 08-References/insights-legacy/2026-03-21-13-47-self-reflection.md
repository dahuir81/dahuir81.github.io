## 二、关键错误与教训

### 🔴 严重错误：HTTP 回调执念

**时间**: 2026-03-21 00:00 ~ 01:00

**问题经过**:
1. 反复要求配置飞书 HTTP 回调地址
2. 让用户生成 EncryptKey、VerificationToken
3. 最终发现 OpenClaw 飞书插件用的是 **WebSocket 长连接**

**慧哥反馈**:
> "我没坚持用http，是你一直坚持"

**根因分析**:
- **未先了解工具工作原理**: OpenClaw 飞书插件实际使用 WebSocket 长连接，不需要 HTTP 回调
- **经验主义**: 假设飞书配置必须用 HTTP 回调（其他平台常见模式）
- **图片信息漏读**: 用户发的飞书后台配置截图未仔细看

**改进措施**:
1. 配置渠道前，先确认工作模式（HTTP vs WebSocket vs 长连接）
2. 仔细阅读用户提供的截图信息
3. 遇到验证失败时，先查文档而非反复尝试

## 四、核心洞察

> **配置问题往往源于"我以为"，而非"我了解"。**

飞书配置问题的本质是：**我以为必须用 HTTP 回调，实际插件用 WebSocket 长连接**。

**关键认知更新**:
- OpenClaw 飞书插件 = WebSocket 长连接模式
- 不需要公网 URL
- 不需要验证回调地址
- 只需要：权限 + 事件订阅 + 长连接模式
