# 养虾逻辑

**类型**: 云平台商业模式

## 定义
「养虾逻辑」是指云平台通过 OpenClaw 吸引用户上云，但真正的收入来自算力消耗而非模型订阅。一旦用户在某云上配置好 OpenClaw 的所有集成，迁移成本往往比订阅费高得多，从而锁定用户。

## 核心逻辑

```
OpenClaw（钩子）→ 吸引用户上云 → 配置集成 → 高迁移成本 → 算力消耗（真正收入）
```

## 各家云平台策略

| 云平台 | 策略 | 核心产品 | 定价 |
|--------|------|---------|------|
| **阿里云** | 卖算力 | 9.9元/月预装 OpenClaw 服务器，PAI-EAS 一键部署 | 9.9元/月起 |
| **火山引擎** | 套餐捆绑 | 订阅 Coding Plan 免费解锁 ArkClaw（云上 SaaS 版） | 40元左右/月 |
| **腾讯云** | 分层覆盖 | WorkBuddy/QClaw → 轻量服务器 → ADP 企业版 | 分层定价 |

## 阿里云：卖算力
- OpenClaw 本身免费
- 但跑 OpenClaw 需要服务器
- 最低 9.9元/月即可拥有一台预装 OpenClaw 的云服务器
- PAI-EAS 提供一键部署专属 OpenClaw
- 可调用百炼平台服务或自定义模型服务

## 火山引擎：套餐捆绑
- 订阅方舟 Coding Plan
- 直接免费解锁 ArkClaw
- 一键云端部署 OpenClaw
- ArkClaw 被描述为「开箱即用的云上 SaaS 版 OpenClaw」
- 解决本地部署环境复杂、配置繁琐、Token 消耗高、进程易中断等痛点

## 腾讯云：分层覆盖
- **个人用户**: WorkBuddy 和 QClaw
- **开发者**: 轻量应用服务器模板，云端秒级启动、7×24 小时连续运转
- **企业客户**: 智能体开发平台（ADP）分钟级部署，原生支持组织架构级权限管理和工作空间数据隔离

## 本质
> "OpenClaw 成了吸引新用户、拉动算力消耗、推动企业上云的「钩子」，算力消耗才是最后真正的收入。"

一旦开发者在某云上配好了 OpenClaw 的所有集成（飞书、Notion、GitHub、本地文件系统等），**迁移成本往往比订阅费高得多**。

## 与 Coding Plan 的关系
- Coding Plan: 按月订阅，不限 Token
- 养虾逻辑: 真正的收入来自算力消耗
- **组合**: Coding Plan 是「钩子」，算力消耗是「真正收入」
