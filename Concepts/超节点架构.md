# 超节点架构 (SuperNode Architecture)

## 核心概念

超节点(SuperNode)是指将大量AI算力卡（GPU/NPU）通过高速互联技术集成在一个或少数几个机柜内，形成"超级计算机"的架构形态。

### 为什么需要超节点？

1. **通信瓶颈**: 传统分布式集群节点间通信延迟高
2. **显存墙**: 单卡显存有限，大模型放不下
3. **扩展效率**: 集群规模越大，线性扩展效率越低
4. **运维复杂度**: 万卡级集群管理困难

## 关键技术指标

### 1. 规模指标

| 指标 | 说明 | 代表产品 |
|------|------|---------|
| 单柜卡数 | 单个机柜内算力卡数量 | NVL72(72卡), 华为384(384卡) |
| 集群规模 | 超节点可扩展的最大规模 | Atlas 950(8192卡), DGX SuperPOD(数千卡) |

### 2. 互联指标

| 指标 | 说明 | 典型值 |
|------|------|--------|
| 互联带宽 | 卡间通信带宽 | NVLink 900GB/s, UB 数百GB/s |
| 延迟 | 卡间通信延迟 | 微秒级 |
| 拓扑类型 | 网络连接方式 | 全连接、Fat-Tree、Dragonfly |

### 3. 内存指标

| 指标 | 说明 | 代表 |
|------|------|------|
| 单卡显存 | HBM容量 | 192GB(H100), 288GB(B200) |
| 共享内存 | 集群统一寻址内存 | Ironwood 1.77PB |
| 内存带宽 | HBM带宽 | 数TB/s |

## 主流超节点架构

### NVIDIA NVLink系列

#### NVL72 (Blackwell)
- **规模**: 72卡/机柜
- **互联**: NVLink 5 + NVSwitch
- **铜缆**: 5184根（含差分对+双工）
- **拓扑**: 无中心、18个NVSwitch、全对等
- **显存**: 1.3TB (72×18GB)
- **扩展**: NVL144/288/576规划中

**5184根铜缆计算**:
```
72卡 × 18 NVSwitch × 2(差分对) × 2(双工) = 5184
```

#### DGX SuperPOD
- **规模**: 数千卡
- **互联**: NVLink + InfiniBand
- **定位**: 企业级AI集群

### 华为CloudMatrix系列

#### 超节点384
- **规模**: 384卡/超节点（多机柜）
- **互联**: UB (Unified Bus) 统一总线
- **拓扑**: 无中心、两层Switch(Layer1/2)、全对等
- **特点**: 全光互联，Switch数量多，通道选择多

#### Atlas 950 SuperPoD
- **规模**: **8192卡**
- **交付**: 2026年下半年
- **定位**: 对标DGX SuperPOD，规模领先

### Google TPU SuperPod

#### Ironwood SuperPod
- **规模**: 9216卡/超算舱
- **互联**: OCS光电路交换机
- **内存**: **1.77PB全局共享内存**（世界纪录）
- **拓扑**: 光互联、全局寻址、共享内存

## 网络拓扑对比

### 华为384 vs NVIDIA NVL72

| 维度 | 华为384 | NVIDIA NVL72 |
|------|---------|--------------|
| 单节点规模 | 384卡 | 72卡 |
| Switch层级 | 两层(Layer1/2) | 单层(18个) |
| 节点关系 | 全对等 | 全对等 |
| 中心节点 | 无 | 无 |
| 互联介质 | 全光互联 | 铜缆+NVSwitch |
| 扩展规模 | 8192卡 | 576卡(规划) |

### 核心洞察

> "华为超节点384与NVIDIA NVL72，从网络拓扑上看几乎是同一种类型，差别主要在规模大小。"

**关键设计哲学**:
1. **无中心**: 避免单点故障
2. **全对等**: 所有节点地位平等
3. **分层/多Switch**: 增加并行通道，减少堵塞
4. **高基数**: Switch端口数决定扩展能力

## 全连接计算

### 连接数公式

```
双工模式: 连接数 = n × (n-1)
单工模式: 连接数 = n × (n-1) / 2
```

### 特征规模连接数

| 节点数 | 双工连接数 | 备注 |
|--------|-----------|------|
| 72 | 5,112 | NVL72 |
| 144 | 20,592 | NVL144 |
| 288 | 82,656 | NVL288 |
| 384 | 147,072 | 华为384 |
| 576 | 331,200 | NVL576 |
| 8192 | 67,092,480 | Atlas 950 |
| 9216 | 84,934,656 | Ironwood |

**趋势**: 节点数增加 → 连接数指数级增长 → 拓扑设计成为核心竞争力

## 技术挑战

### 1. 互联密度
- 节点越多，互联线缆/光纤数量爆炸
- 物理空间、散热、布线复杂度

### 2. 一致性
- 缓存一致性协议复杂度
- 内存一致性保证

### 3. 故障隔离
- 单点故障影响范围控制
- 热插拔、在线替换

### 4. 软件生态
- 分布式训练框架适配
- 通信库优化(NCCL等)
- 编程模型抽象

## 未来趋势

### 1. 规模持续扩大
- 从72卡 → 384卡 → 8192卡 → 万卡级
- 单集群训练万亿参数模型

### 2. 光互联普及
- 铜缆带宽瓶颈
- 光互联低延迟、高带宽、低功耗
- CPO（共封装光学）技术

### 3. 共享内存架构
- 全局统一寻址
- 打破显存墙
- 简化编程模型

### 4. 存算一体
- 计算靠近存储
- 减少数据搬运
- 3D堆叠技术
