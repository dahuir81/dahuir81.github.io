# 2026-04-13 QQ邮箱SMTP授权码泄漏事件

## 事件概述

今早发现 QQ 邮箱（5283728@qq.com）收到大量退信，检查发现从 **06:47** 开始有异常邮件发送，**08:19** 后爆发式群发，共 **227 封**异常邮件。

## 根因定位

**泄露源头**：`~/.openclaw/workspace/blog/notify.sh` 中硬编码了 SMTP 授权码，且该文件被 commit 并 push 到了公开 GitHub 仓库 `dahuir81/dahuir81.github.io`。

```python
# notify.sh 中的泄露代码
smtp_server = "smtp.qq.com"
smtp_port = 587
sender_email = "5283728@qq.com"
password = "ckyfpclygvajbjgf"   # ← 授权码明文硬编码
```

该文件通过博客发布工作流（blog-publish-workflow）被创建，用于博客部署成功后发送邮件通知到 5283728@qq.com。

## 攻击时间线

| 时间 | 事件 |
|------|------|
| 06:47 | 第一封测试邮件，收件人 `claudineviville8@gmail.com`，主题 `Test de validation SMTP` |
| 08:19 | 开始爆发式群发，主题 `Vous avez (1) nouveau message en attente`（法语钓鱼模板） |
| 09:46 | 收到第一封退信（来自 PostMaster@qq.com） |

## 攻击特征

- 发送源 IP `98.81.133.132`（美国弗吉尼亚州），非本地 IP
- 邮件内容为法语钓鱼模板，收件人为全球随机邮箱地址（yahoo.in, gmail.com, hotmail.fr 等）
- 攻击者通过泄露的授权码直接连接 `smtp.qq.com` 群发，未通过 macOS Mail 客户端
- 邮件头：`Received: from [127.0.0.1] ([98.81.133.132]) by newxmesmtplogicsvrszb43-0.qq.com (NewEsmtp)`

## 排查过程

1. 检查 macOS Mail 应用配置，确认本地未主动发送
2. 检查本地发件箱（Outbox）和已发送目录，确认异常邮件均通过 `smtp.qq.com` 远程发送
3. 排查 OpenClaw 全目录（openclaw.json、credentials、workspace、logs、agents/sessions），搜索 `smtp`、`mail`、`授权码` 等关键词
4. 定位到 `~/.openclaw/workspace/blog/notify.sh`，确认硬编码授权码
5. 确认该文件已推送到 GitHub 公开仓库 `dahuir81/dahuir81.github.io`

## 经验教训

- **绝不在代码仓库中硬编码任何密码/授权码/Token**，即使是私人仓库也存在风险
- GitHub 公开仓库中的泄露会被自动化爬虫快速扫描到
- `git filter-branch` 可从历史中清除文件，但泄露窗口期内可能已被爬取，事后补救有限
- 授权码泄露后应立即全部停用，不能只停可疑的那一个
- 邮件通知等自动化流程应使用平台 Secret 机制（GitHub Actions Secrets / OpenClaw credentials）
- 本地 `.env` 或配置文件中的敏感信息也应排除在 git 跟踪之外
