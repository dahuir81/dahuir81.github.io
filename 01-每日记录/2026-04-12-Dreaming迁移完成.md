# 2026-04-12 Dreaming 迁移完成

## 背景
- 百炼云端记忆（modelstudio-memory-for-openclaw）即将收费，之前评估过两个系统不能共存
- 目标：完全启用 OpenClaw 内置 Dreaming 系统，体验自动记忆整理能力
- 同时将已有 Obsidian 知识库体系融入 memory 搜索

## 执行过程

### 阶段 A：技术调研
- 确认 memory slot 机制：一次只能一个插件占用
- 评估三种 Obsidian 知识库融合方案，结论：
  - 方案A（workspace重定向）：风险太大，影响 AGENTS.md 等加载
  - 方案B（symlink桥接）：memory-core 显式拒绝 symlink，不可行
  - 方案C（extraPaths）：可行，纳入搜索但不参与 Dreaming 整理

### 阶段 B：实施
1. 备份到 `~/.openclaw/backup/2026-04-11-pre-dreaming/`
2. 修改 `plugins.slots.memory` → "memory-core"
3. 禁用百炼 memory 插件
4. 配置 memory-core Dreaming（03:00 每日执行）
5. 添加 extraPaths 索引 Obsidian 知识库
6. 网关重启验证

### 遇到的问题
- **memory-core 不加载**：第一次重启后 gateway.log 显示 "ready (3 plugins)"，没有 memory-core 日志
- **根因**：SIGTERM 后的第一次启动插件计数不准确，但通道完全启动后 memory-core 确实加载了
- **修复**：实际是正常工作的，gateway 日志后续出现了 `memory-core: created managed dreaming cron job.`
