Executive Summary

阿里云能否通过 token 消耗形成可持续的 AI 云商业闭环？

初步判断：可以形成，但不是“单靠 token 单价赚钱”的闭环。更稳健的闭环是：token 作为推理用量计价单位和开发入口，带动模型 API、企业套餐、专属推理、训练微调、数据库、存储、云原生和行业解决方案的组合收入。

核心判断
有条件成立：token 是入口与计量层，利润池更可能来自 AI 云全栈。

支撑证据包括：阿里云 2026 财年云智能集团收入 1,581.32 亿元，同比增长 34%；2026 年 3 月季度云收入 416.26 亿元，同比增长 38%；AI 相关产品收入 89.71 亿元，且官方称占云外部收入 30%。同时，百炼 Model Studio 客户数同比增长 8 倍。[S1]

模型能力

Qwen 系列模型提供调用理由：代码、推理、长上下文、多模态和行业 Agent 能力。模型能力越稳定，客户越愿意把更多任务迁入 API 或百炼工作流。

  • 收入：高级模型 token、企业套餐、专属模型服务。
  • 成本：训练、后训练、推理优化、模型安全和评测。
  • 锁定：提示词、评测集、工具链、业务流程迁移成本。
Strategic Context

战略背景：阿里云已经把 AI 云从“试验项”推到收入主线

最新财报的措辞已经从“投入 AI 能力”转向“规模化商业化回报周期”。但利润端仍需拆开看：云 EBITA 改善，不等于 token API 自身已经高毛利。

Monetization Mechanism

Token 到底变现的是什么

token 不是商品本身，而是把模型能力、推理算力、服务质量和客户业务价值压缩成可计量、可结算、可治理的单位。

1. 推理服务

客户购买的是“把请求转成可用答案”的生产能力。输入 token 代表读取上下文的成本，输出 token 通常消耗更多推理时间，因此输出价格更高。

2. 工程抽象

百炼把模型选择、OpenAI 兼容接口、监控、告警、密钥、限流、日志和计费收进一个平台，降低客户自建推理栈的复杂度。

3. 企业确定性

Token Plan、Coding Plan、专属部署和 PTU/资源包把不确定的用量转成预算、配额、性能隔离和采购流程。

关键判断：token API 更像“水龙头”，水价能收钱，但真正的商业价值在于水网、压力、稳定性、接入设备和长期合同。阿里云的优势在水网，风险在水价。

Revenue & Cost Model

收入与成本模型：不要把 AI 收入误读成纯 token 收入

官方披露的 AI 相关产品收入很大，按当前 API 单价反推，需要极高 token 量。因此更合理的解释是：AI 相关产品收入包含模型调用、训练推理算力、平台服务、专属实例和周边云资源。

阿里云相对优势

中国云基础设施领导地位、Qwen 模型体系、百炼 MaaS 平台、PAI/EAS/灵骏承接训练推理、集团电商和钉钉等应用场景可作为内部需求和示范场景。Canalys 报道显示，2025 年一季度中国云基础设施支出中阿里云份额约 33%，仍居第一。[S14]

阿里云主要短板

低价模型持续压制 API 溢价；企业客户可能要求多模型中立；AI 基础设施 CapEx 与芯片供给约束抬高成本；模型开源和私有化部署会把部分高频需求从公有 API 抽走。

Conditions & Risks

什么时候成为增长引擎，什么时候变成价格战

下面不是结论清单，而是判断闭环是否成立的条件树。每个条件都应该能在未来财报、产品定价或客户案例中找到证据。

闭环成立的条件

模型能力持续进入第一梯队。 企业不会为低质 token 长期付费，代码、推理、多模态和 Agent 成功率要足以支撑生产场景。

推理成本下降快于价格下降。 Batch、缓存、调度、MoE、量化、自研或国产芯片适配必须把单位成本压低。

token 带动周边云消费。 RAG、向量检索、数据库、OSS、EAS、日志和安全治理形成复购，才有云厂商优势。

从开发者转向企业采购。 Token Plan、专属实例、行业 Agent 和 SI 生态把偶发调用变成预算内持续支出。

闭环断裂的风险

API 被商品化。 如果客户只比较每百万 token 价格，阿里云会被 DeepSeek、火山、百度、开源自部署压价。

PoC 无法规模上线。 调用量看似增长，但业务价值无法证明，企业会削减预算或改为私有化低频使用。

算力成本与价格错配。 基础设施价格上涨、GPU/内存供给紧张、利用率不足，会让 token 增长带来利润压力。

平台锁定不够强。 OpenAI 兼容接口降低迁移成本，也让多模型路由更容易替代单一云平台。

Watchlist

未来 1-3 年关键观察指标

建议把这些指标分为“增长是否真实”“毛利是否可持续”“锁定是否形成”三类跟踪。
1
AI 相关产品收入占云外部收入比例
是否从 30% 继续提升，同时云 EBITA 率不恶化。
2
百炼付费客户和 ARPU
仅有客户数增长不够，需要付费客户、复购、资源包和企业席位扩张。
3
价格与成本曲线
Qwen、DeepSeek、豆包、文心、混元的价格变化；阿里是否更多使用缓存、Batch、套餐保持毛利。
4
推理性能指标
TTFT、TPS、失败率、TPM/RPM 保障、上下文缓存命中率和峰值稳定性。
5
周边云 attach rate
AI 客户是否同时购买 PAI/EAS、数据库、OSS、网络、安全和观测服务。
6
CapEx 与自由现金流
云基础设施支出是否转化为收入增长，而不是形成长期现金消耗。
7
行业案例从 PoC 到生产
医药、金融、制造、零售、跨境贸易等是否有可量化 ROI，而非发布会案例。
8
私有化与专属部署比例
若大客户持续转向专属算力，token API 仍是入口，但收入确认和毛利结构会更像传统云。
Conclusion

结论与讨论问题

最终判断不是“看多”或“看空”，而是明确：哪些证据说明闭环正在形成，哪些反证会推翻它。

最终判断

阿里云 token 变现逻辑有条件成立。短期看，token 能作为 MaaS 和开发者入口推动 AI 相关产品收入增长；中期看，闭环能否可持续，取决于 token 是否转化为企业生产系统里的持续工作负载，以及这些工作负载是否带动 PAI、EAS、数据库、存储、网络、安全和专属算力。

最大风险是把 token 视为独立利润池。国内模型 API 价格会长期承压，低价模型和开源部署会持续存在。阿里云真正的可防守位置，是“模型能力 + 推理效率 + 云资源 + 企业治理 + 生态场景”的组合，而不是某一个 Qwen API 的标价。

关键反证

如果未来 4-6 个季度出现“AI 相关收入继续增长但云 EBITA 率下降、自由现金流压力扩大、API 价格持续下调、企业案例停留在试点、Token Plan 主要靠低价吸引开发者”的组合信号，应下调闭环成立判断。

上修信号

如果阿里披露更多生产级客户、AI 相关产品占比继续提升、Model Studio 付费客户和企业席位扩张、云 EBITA 率稳定改善、专属推理和周边云资源 attach 增强，应上修判断。

建议讨论问题

  1. 我们讨论的 token 收入，是开发者 API 收入，还是 AI 相关云产品收入？口径必须先统一。
  2. 阿里云应优先追求 token 市占率，还是优先维护云 EBITA 和现金流？
  3. Qwen 的优势场景是代码、办公、行业 Agent、消费应用，还是通用推理？哪个场景最有付费纵深？
  4. 低价模型出现时，阿里应降价跟随、用套餐锁定，还是转向专属部署和周边云资源？
  5. 企业客户何时会从 API 调用迁移到专属实例或私有化？这对毛利是利好还是利空？
  6. 如果未来披露 token 调用量，我们应重点看总量、有效输出 token、缓存命中率，还是单位收入？
