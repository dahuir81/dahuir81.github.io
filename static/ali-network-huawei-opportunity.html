1️⃣ 阿里 TPN：Token 生产网络架构

概念首提

蔡德忠 · CCF 秀湖会议（2026年4月）

阿里云在 CCF 秀湖会议上首次提出 TPN（Token Performance Network），核心主张：推理时代的网络不再围绕 GPU/Job 定义，而要围绕 Token 生产效率定义。

目标函数从"峰值带宽"转向"极致 Token 性能与最低 Token 成本之间的系统最优解"。推理网络的终局是 Token Economics——不是比谁的带宽大，而是谁能用更低成本持续产出更多有效 Token。

推理时代为什么网络成了主角？

PD 分离

KV-Cache 网络化

Prefill-Decode 分离把 KV-Cache 从本地缓存推到网络，推理从 compute-bound 坠入 network-bound。

Agentic

高复用缓存命中

Agent 场景 KV-Cache 命中率 95%+，瓶颈从"算"转向"取"——推理系统成为网络吞吐瓶颈的奴隶。

MoE 风暴

Expert 路由小包风暴

MoE 架构下 Expert 路由产生持续微秒级小包风暴，对网络拥塞控制和 QoS 提出极端要求。

跨域协同

跨集群状态接续

Agent 长时间运行需要跨集群/跨域的状态迁移和接续，广域网成为推理网络的自然延伸。

2️⃣ 阿里 HPN：高性能训练网络

产品

阿里 HPN 网络架构

阿里 HPN（High Performance Network）是阿里云面向 AI 训练场景设计的高性能网络架构，核心特征如下：

  • 双 ToR + 双平面：采用冗余 TOR 交换机 + 双平面设计，提升可靠性和带宽
  • 规模上限：最多连接约 15,000 GPU（对比下一代 MRC 架构可达 100K+）
  • 定位：面向中大规模训练集群，已验证在阿里云内部大规模部署
  • 演进方向：HPN 7.0 引领智算以太网生态，向高通量以太网和 ENode+ 超节点演进

关键观察：阿里 HPN 在训练侧采取了稳健的双平面架构，但相比 OpenAI MRC 的 100K+ 规模和华为超节点的高密度互联，规模扩展性和创新密度有差距。阿里正在通过平头哥磐脉 920 智能网卡（内置 PCIe Switch、400Gbps）和 TPN 概念补足网络能力。

⚡ Token 时代路线对立：硬件 vs 软件

🔧 硬件/基础设施派

蔡德忠：硬件实现更高效

Token 生产的确定性时延和超大规模吞吐只能靠专用芯片、物理层创新和高通量以太网协议解决。软件抽象固然带来灵活性，但牺牲了不可接受的性能和确定性。HPN 7.0 和 ENode+ 超节点都是硬核硬件方案。

核心论点：推理网络需要按"链路角色"差异化承载（TTFT / TPOT / KV / Expert 各走各路），这种差异化只能在物理层实现——硬件交换机做微秒级 QoS 调度才是真正答案。

💬 "X10 规模的智算集群，Ethernet 将成为主流方案"—— CCF ChinaNet 2024

⚙️ 软件/虚拟化派

祝顺民：软件实现更有效

Token 流的多样性（TTFT / TPOT / KV / MoE 各路流量特征完全不同）恰恰需要 SDN 层的智能调度和虚拟化抽象。硬件方案做得再好，本质上是一套固定的策略——而 Token 时代流量模式在持续演变。阿里云的规模决定了软件自动化是唯一出路。

核心论点：硬件确定性在网络规模和多样性到达一定程度后必然让位于软件的弹性。洛神平台 10 年验证了软件定义可以承载百万级用户和 8 次双 11——token 时代也跳不出这个规律。

💬 "软硬一体虽然在性能上优势明显，但云需要被软件定义"—— 洛神平台技术理念

💡 对华为合作策略的启发

  • 蔡德忠路线 → 华为的强连接点：交换机硬件合作（CloudEngine + 高通量以太网互认证）、光模块供应、超节点架构经验输出。华为的硬件基因与蔡德忠路线天然匹配，容易切入。
  • 祝顺民路线 → 华为的弱连接点：SDN 控制器互操作、云网协同联合验证、5G/6G 云专网标准。华为 iMaster NCE 与洛神存在对接可能，但软件层合作更敏感。
  • 当前谁占上风：从公开曝光度看，蔡德忠路线（HPN 7.0 / 高通量以太网 / TPN）获得了阿里云更高调的战略背书；但洛神平台是阿里云收入的压舱石，祝顺民在收入贡献上话语权更实。
  • 建议接触优先级：光互联合作（硬件采购）→ 交换机互认证（硬件合作）→ 超节点架构交流（技术互鉴）→ SDN 互联互通（软件标准层面，敏感度最高放最后）。

4️⃣ 与华为合作机会：网络领域切入点

🎯 核心判断

阿里云正经历从"训练基建"到"推理基建"的重大转向，TPN 是其推理网络的新理论框架。华为在网络设备（交换机/路由/光互联）和超节点架构上有深厚积累。双方在网络基础设施层面存在互补性，但需注意华为也自研昇腾 AI 平台的竞争关系。

方向一：推理网络交换机与互联

阿里需求

推理网络差异化承载

TPN 要求按 TTFT、TPOT、KV、Expert 路径做差异化 QoS。阿里需要支持微秒级调度、多平面隔离、确定性时延的交换机芯片和整机设备。

华为能力

CloudEngine + 自研芯片

华为 CloudEngine 系列交换机+自研昇腾/鲲鹏网络芯片，支持 RoCEv2、智能无损网络、微秒级拥塞控制。其在运营商和企业网的高端交换机能力可复用。

方向二：CXL 互联与内存池化

阿里需求

KV-Cache 池化硬件

TPN 的 KV-Cache 网络化需要 CXL Switch 和 CXL 内存池化硬件。PolarDB 已验证 CXL Switch 方案，但推理侧的 KV-Cache 池化需要更大规模部署。

华为能力

CXL 与内存互联

华为在内存池化、CXL 协议实现和高速互联方面有技术储备（鲲鹏处理器内置 CXL 控制器），可与阿里 PolarDB/TPN 场景形成互补。

方向三：400G/800G 光互联

阿里需求

大规模集群光互联

阿里云数据中心正在部署 400G，向 800G 演进。HPN 和未来 MRC 架构都需要高性能光模块和光互联方案。

华为能力

光产业全栈

华为海思光电子在高速光模块、硅光技术上有深厚积累，是少数能提供从光芯片→光模块→光互联系统全栈方案的厂商。

方向四：智能网卡与 DPU

阿里需求

RDMA 卸载加速

阿里平头哥磐脉 920 已实现 400G 智能网卡（内置 PCIe Switch），但面临 NVIDIA ConnectX-8 和 AMD 方案的竞争。需要更丰富的网卡生态支撑。

华为能力

鲲鹏智能网卡

华为鲲鹏智能网卡和昇腾 AI 算力网的集合通信方案已经过大规模验证。RDMA over Converged Ethernet 方面华为有深厚积累，可与阿里 TPN 的推理流量卸载需求对接。

📌 讨论策略建议

  • 先易后难：从光互联（采购/合作）切入，建立合作基础
  • 技术互鉴：阿里 TPN 的理念可以为华为推理网络设计提供思路，华为超节点的高密度互联经验对阿里有参考价值

  • 看人下菜碟：硬件层面的合作（交换机/光模块）对蔡德忠路线，软件层面的互鉴（SDN/调度平台）看祝顺民路线的走向
