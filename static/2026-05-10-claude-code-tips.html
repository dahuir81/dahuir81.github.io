1 四条黄金规则（CLAUDE.md）

Karpathy 最佳实践 + 实战经验，写在项目根目录的 CLAUDE.md 里，Claude Code 每次启动自动读取。

🧠

规则一：先想，再写

动手之前先确认需求。不猜、不假设、不自己加默认值。

  • 需求不明确时，停下来问用户，不要自己脑补
  • 列出你需要确认的问题清单，等用户回答再动手
  • 不确定的地方明确说出来，而不是藏起来假装懂了
✂️

规则二：能简单就简单

  • 能一行解决的不要写十行
  • 能一个文件搞定的不要拆五个文件
  • 200 行能缩成 50 行的，重写
  • 没被要求的功能不加，没被要求的灵活性不给
检验标准：资深工程师看了觉得刚刚好，不是"这也太复杂了吧"
🎯

规则三：只改该改的

  • 只改用户要求改的部分，旁边的代码再丑也不碰
  • 注释过时？不改。风格不同？照着原来的写。命名不规范？不动。
  • 每一行改动都必须能追溯到用户的需求
🏁

规则四：目标驱动，别指令驱动

  • 给成功标准，不给步骤
  • 用户说"修这个 bug"，你就写能复现 bug 的测试然后让它通过
  • 用户说"加个功能"，先问清楚"做到什么程度算完成"
强的成功标准让 AI 自己闷头跑，弱的只会越跑越偏

2 Code with Claude 大会四大新功能

2026 年 5 月 7 日 Anthropic 开发者大会发布，Claude Code 从「跑几分钟」进化到「连续跑几个小时」。

🌙 Dreams 研究预览

Agent 空闲时自动复盘历史对话，扫描反复出现的错误，提取有效工作模式，存入记忆。多 Agent 场景下汇总团队经验库——"一起做梦，一起变聪明。"

✅ Outcomes 公测

定义交付质量标准，独立评分系统打分，不达标重来。Dreams 管经验积累，Outcomes 管质量把关。

⚡ Routines 公测

设置好需求，Claude Code 在后台自己执行，完成后直接生成可合并的 PR。不用盯着，干完了通知你审核。

📱 Remote Agents 公测

手机远程控制笔记本上的 Claude Code。出门在外也能指挥 Agent 干活。

🔍

ultrareview（云端代码审查）

一群 Agent 在云端集体审查代码，发现问题推送回终端。不是单个 Agent 看一遍，是多个 Agent 各自审查后汇总。

🏗️

Multi-agent Orchestration

复杂任务拆分给多个 Agent 并行处理协同交付。Boris Cherny 同时跑着几十个 loop，每晚几千个 Agent 做深度异步任务。

3 Agent Loop：让 AI 自己跑起来

"他越来越觉得 loop 就是未来" — Boris Cherny, Claude Code 创建者

Claude Code 的核心运行机制 Agent Loop 只有几十行代码，可拆解为 11 步，重点在第 5-8 步：Agent 接到任务后进入循环判断——需要做什么、需要什么上下文、是否要调用工具、任务是否完成——如果未完成继续执行。正是这个机制让 Agent 从处理 1 分钟任务跨越到 2 小时以上。

Boris Cherny 的实战配置

Loop 任务频率说明
监控 PR 状态实时自动 rebase、合并
维护 CI持续发现 flaky test 就去修
抓取反馈每 30 分钟从推特抓 Claude Code 反馈，聚类后推送
深度异步任务每晚几千个 Agent 批量处理
Anthropic Routines 已把 Loop 做成服务端功能——关掉电脑，任务照样跑。

5 HTML 工作流：md 生产，html 消费

Claude Code 工程师 Thariq 发文引爆讨论，花叔提出综合框架：md 和 html 不是替代关系，是分工关系。

为什么？AI 解耦了生产和消费

MarkdownHTML
定位生产端——给 AI 写消费端——给人看
优势轻、快、可 diff、token-efficient丰富、可视化、可交互、好分享
Token3,15016,180
谁写你 + AIAI
谁看你（内部 review）别人（分享/展示）
📊

Cloudflare 实测

同一篇博客：HTML 16,180 token，md 只要 3,150 token。压缩 80% token → 同样 LLM 预算处理 7-17 倍请求。

以前 vs 现在

以前

你写 md，看 md。格式要折中——md 胜出。

→

现在

你写 md，改 md，AI 给你 html。两端各自登顶。

Thariq 的五大 HTML 场景

  1. 发散性思考：每个方向扩展为独立 HTML 页面，另一个 Agent 直接"读整个 HTML 网络"
  2. 代码审查：HTML 渲染 diff + 行内批注 + 模块关系图，比 GitHub 默认视图强一个量级
  3. 设计原型：Claude 先用 HTML 勾勒高保真设计图，再转译为目标语言
  4. 报告输出：整合多数据源，生成交互式 HTML 文档
  5. 自定义编辑器：为特定数据构建一次性 HTML 编辑界面
🔧

花叔的 huashu-md-html 工具

开源 Claude Code Skill，三个能力组成完整闭环：

  • 任意格式 → md：封装 markitdown，20+ 格式一行命令
  • md → 精美 html：Pandoc + 4 套 CSS 主题（article / report / reading / interactive）
  • html → md：双引擎（trafilatura + markitdown）

GitHub: github.com/alchaincyf/huashu-md-html · MIT License
