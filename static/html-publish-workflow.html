🔁 标准工作流

1

慧哥发起需求

在钉钉直接说"出一份简报/分析"，附上背景信息或来源链接。

2

Tars 生成 HTML

写文件到 ~/Documents/GitHub/blog/static/[文件名].html，自包含样式，无外部依赖。

3

运行发布脚本

publish-html.sh [文件名].html → Git API 推送 → GitHub Actions 构建 → 上线

4

ECS 自动同步

阿里云 cron 每 15 分钟拉取 GitHub Pages 最新内容到 openclawmy.work

5

归档 & 更新索引

KnowledgeBase 归档 + 更新分享目录索引文件 + 更新索引

6

发送可点击链接

钉钉单独发送两条纯文本链接（GitHub + ECS），不混在文字里，不用卡片。

自动化

每次发布自动更新

在 publish-html.sh 末尾追加一条归档逻辑：

  • 提取 HTML 的 <title> 作为页面标题
  • 按日期归类，追加到 08-参考/发布清单/README.md
  • 生成一个独立的 08-参考/发布清单/索引页.html，包含所有历史链接
  • 这个索引页本身也可以发布到 GitHub 供外部访问
