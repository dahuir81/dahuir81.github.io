## Foundry 平台架构

### Context（上下文工程）

**公式**: Context = 数据 + 权限 + 行动 + 记忆

| 维度 | 功能描述 |
|------|---------|
| **数据** | 将 SQL 表转化为业务对象（如"飞机"、"航班"），通过知识图谱关联 |
| **权限** | 继承源系统数据权限，确保合规访问 |
| **行动** | 通过 API 调用源系统能力（如 CRM 发优惠券） |
| **记忆** | 记录每次执行结果，持续优化策略 |

### Harness（马具工程）

- **规则校验**: AI 输出必须经过业务规则验证（如授信额度检查）
- **测试环境**: 真实数据压测，控制"幻觉"风险
- **Human-in-the-loop**: 关键决策需人工确认

## 核心洞察

> "当上帝掷下大模型这颗火种时，只有 Palantir 准备好了燃料。"

**Palantir 逆袭的本质**:
- **23 年长期主义**: 别人做"轻量化、快部署"，Palantir 做"重交付、底层数据治理"
- **企业级上下文**: 掌握 "我的公司" 的数据、权限、流程
- **大模型补齐拼图**: 提供了交互入口，让复杂的 Foundry 变得易用

**商业模式**:
- 不卖模型，卖 "AI 操作系统"
- 客户无需折腾模型，接入 Foundry 即可让 AI 开始管仓库
