## 华为竞争策略

### "超节点+集群"战略

**核心逻辑**（徐直军）：
> "基于中国可获得的芯片制造工艺，我们努力打造'超节点+集群'算力解决方案，来持续满足算力需求"

**三大支柱**:
1. **规模效应**: 极致集群化，数千甚至数万颗芯片协同
2. **通信优化**: 灵衢协议降低通信时延到微秒级，带宽提升到PB级
3. **能效平衡**: 全液冷设计，PUE≤1.15

### 产品分化策略

| 芯片 | 定位 | 内存 |
|------|------|------|
| **Ascend 950PR** | Prefill和推荐系统 | HiBL 1.0（低成本） |
| **Ascend 950DT** | Decode和训练 | HiZQ 2.0 HBM（高性能） |

### 架构创新

**SIMD+SIMT混合架构**:
- SIMD: 处理结构化连续数据（Prefill）
- SIMT: 处理碎片化非结构化数据（Decode）

**内存访问优化**: 从512字节精细化到128字节，提高带宽利用率

## 灵衢2.0深度解析

### 设计哲学

从"CPU中心"到"全平等互联":
- 传统: CPU调度所有数据交换
- 灵衢: 数万计算部件横向扩容，接近内存总线的高带宽

### UB-Mesh拓扑

**nD-FullMesh递归直连**:
- 1D: 单板内8个NPU FullMesh
- 2D: 单机架内64个NPU + 8个CPU Mesh互联
- 3D/4D: 跨机柜同层或跨楼层全连接

**扩展**: 以64卡为步长，最大8192卡无收敛全互联

### 协议归一

**URMA（Unified Remote Memory Access）**:
- 支持同步Load/Store、异步读/写、原子操作、消息传递
- 整合分散内存为统一虚拟内存空间（SvM）

### 性能突破

| 指标 | 数值 |
|------|------|
| 同步内存语义时延 | 百纳秒级 |
| 异步内存语义时延 | 2~5微秒 |
| 单UB通道数据速率 | 14GB/s（112Gbit/s） |
| 单UB链路带宽 | 224GB/s（1.8Tbit/s） |
| 系统级互联带宽 | 16.3 PB/s |
| 跨柜时延 | 3微秒（传统7微秒） |
