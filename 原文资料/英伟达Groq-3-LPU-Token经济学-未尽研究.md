# 当token经济学向低延迟倾斜，中国开源生态短板浮现

推理的token经济学，是一条吞吐量（Throughput，TPS/兆瓦）与交互性（Interactivity，TPS/用户）之间展开的价值曲线。

在GTC 2026上，黄仁勋拿起最新的Groq 3 LPU，把低延迟从性能指标，转化为能够直接重塑单位token价值的变量。在他的演示中，当交互速度从约50TPS（tokens/秒）提升至800TPS甚至以上，单位token所对应的价值，从几美元跃升至"150"美元。

英伟达当代算力平台GB NVL72，是模型经济性测评InferenceX的"推理之王"，但仍然止步于400TPS。黄仁勋试图进一步让市场相信，即将到来的Vera Rubin与Rosa Feynman平台将突破这一天花板。主导该测评的Dylan Patel坐在场内，看着前排一位观众买入英伟达股票，并将交易截图和现场照片发到微信上。

当这条曲线向"交互性"一侧延伸，面向实时反馈的智能体的应用场景开始浮现。而这正是中国开源生态的软肋。量大管饱的Qwen-3模型，已经相当于真正的水与电，但那些高价值的高速交互需求，仍然缺乏对应的算力供给。

## 推理系统的核心约束

推理系统的核心约束，是吞吐量与交互速度之间的此消彼长。批处理（batching，即同时处理更多用户请求）可以提升总吞吐，但会拉长单个请求的响应时间；而降低延迟，则意味着牺牲系统的整体服务能力。在经济上，推理成本按时间计费，而收入按token计算。理论上，在给定的算力约束下，系统在单位时间内能够生成的token总量是相对固定的，高吞吐（低单位价值）与低延迟（高单位价值）只能沿着这条边界进行权衡。

去年的GTC上，英伟达发布的Dynamo，主要解决的就是如何"更有性价比地生产token"的最优函数问题。推理有时候需要优化延迟，有时候需要优化吞吐量。这都得靠一个叫做Dynamo的操作系统来协调。

但现实的问题在于，推理的应用场景并不会均匀分布在这条"权衡曲线"上。实时语音、自动驾驶以及多智能体交互，往往对延迟极为敏感；而普通对话或视频生成，则可以容忍更高的延迟。这些场景的规模与用户的付费意愿，同样呈现出高度不均衡的分布。而当前新增与扩张最快的应用，恰好集中出现在低延迟一侧。

当AI正在能替人类编程数个小时甚至数天，编程的速度就愈发关键；当多个智能体协作完成更加复杂的任务，延时就变得更加不能容忍。年初，OpenAI推出Codex-Spark，实现了即时编程，速度达到1000token/秒。而在更广泛的白领工作中，几乎实时的反馈能够让用户持续停留在"心流"状态。这种不被打断的连续性本身就是高价值的工作体验。

人类之所以更容易容忍与ChatGPT对话的延迟，本质上受限于人类自身的交互窗口，即眼睛的阅读速度与大脑的处理速度。而在智能体之间的交互中，这种速度则显得近乎"冰川级缓慢"。在英伟达hyperscale业务副总裁Ian Buck的描绘里，即将到来的多智能体系统，将运行在1500 TPS甚至更高的尺度之上。未来的软件，主要是给几乎无延迟的智能体用的，那才是AI最前沿的机会所在，英伟达正在创造一个足够支撑"非人类节奏"的生态系统。

## Groq 3 LPU：英伟达的第七块芯片

随着应用与价值不断向低延迟一侧集中，仅依赖模型与软件优化，已经难以突破吞吐与延迟之间的权衡框架。要真正把这条曲线整体向"高交互性"方向推移，仍然需要来自算力硬件层面的跃迁。这正是英伟达此次试图用Groq重写token经济学的关键。

很长一段时间，超低延迟推理是一个"利基"市场，由Groq与Cerebras等少数芯片厂商主导。但去年拐点出现，这一市场开始从边缘走向中心。

去年年底，英伟达向Groq支付"200"亿美元，获得其推理技术非排他性许可。Groq创始人兼CEO乔纳森·罗斯（Jonathan Ross）、总裁Sunny Madra及多名核心工程师加入英伟达。"3"个月后，英伟达的Rubin平台终于迎来了第"7"款芯片Groq 3 LPU（语言处理单元），而且进入批量生产阶段，由三星代工。

Groq 3 LPU在算力方面的性能相对有限，仅为Rubin GPU的1/25。不过，它拥有庞大的500 MB SRAM缓存，带宽达到150TB/秒，远超Rubin的22TB/秒。这使其在低吞吐量但低延迟工作负载中具有特殊优势。基于Groq 3 LPU，英伟达打造了Groq 3 LPX机架，拥有"256"个LPU芯片，专为低延迟、长上下文的智能体系统设计。它将于今年下半年推出。

## 从CPX到LPX：架构转向

这一转向，与半年前英伟达的设计形成了鲜明对比。在此前的思路中，推理流程中的算力密集的预填充阶段（Prefill）与内存密集的解码（Decode）阶段逐步硬件解耦。Rubin CPX正是一款专门针对预填充计算优化的定制芯片。它取消了昂贵的HBM，代之以GDDR7内存，以降低整体单位token成本。而生成与解码计算仍然由旗舰的Rubin GPU负责。

随着LPX的推出，CPX没有出现在演讲现场。相比之下，基于Groq的LPX平台，如今才是英伟达推理架构中的关键拼图。它本质上是针对特定解码阶段的定制优化。在这一架构中，解码过程被进一步拆分，对算力要求更高的部分仍由Vera Rubin承担；预填充也全部交给Vera Rubin。LPU则负责以feed-forward为主的计算，以及token生成等对延迟高度敏感的路径。两者通过Dynamo统一调度，在优化通信机制下协同运行，将整体延迟进一步压缩。黄仁勋建议LPX配比在25%左右。

此外，从CPX转向LPX，也在现实层面绕开了先进封装与高带宽内存的产能瓶颈。无论是台积电的CoWoS，还是SK海力士与三星的HBM供应，都面临着供给不足，价格上涨的局面。在这样的约束下，即便是以DDR替代HBM来压低成本的路径，也不再具备足够的性价比。

作为英伟达的"首席销售官"，黄仁勋友情提示市场，相比单一的Rubin服务的市场，LPX额外打开了"1500"亿美元的增量市场。

## 竞争格局：英伟达并非高枕无忧

不过，英伟达并未完全打动市场。Groq 3 LPX在架构上仍延续此前的LPU C2C（芯片直连）互联，尚未过渡到NVLink体系；对更低精度NVFP4的支持，也需等待后续的Groq 3.5（LP35），而真正由英伟达深度参与设计的LP40，则要到Feynman时代才会落地。

与此同时，竞争对手并未停下脚步。Codex-Spark是OpenAI首个运行在英伟达GPU竞争对手Cerebras芯片上的模型。上周，亚马逊AWS亦宣布与Cerebras合作，打造类似LPX的推理平台。其中，Trainium专注于预填充工作，Cerebras WSE根据结果专门执行解码。在新闻稿中，Cerebras宣称已支持OpenAI、Cognition与Meta模型，速度最高可达3000 tokens/秒。而微软的Maia 200，也转向了以大规模片上SRAM为核心的设计路径。

相比之下，黄仁勋此前高呼来自中国芯片厂商的竞争压力，但在低延迟推理这一关键路径上，仍难看到足够有力的回应。与此同时，中国开源模型的token消耗规模，正在快速逼近美国。无论是字节跳动与谷歌第一方模型日均调用量的接近，还是在OpenRouter等平台上，MiniMax、阶跃星辰等模型消耗占据前列位置，都支撑起"token出海"的叙事。

但也是在这一刻，中国开源生态在这条价值曲线上的处境略显尴尬。在黄仁勋所展示的坐标上，中国的开源模型最快推理速度在每秒100个token，每百万token的价格从免费到3美元不等，而美国的闭源模型的推理速度将很快从每秒400token向1000token延伸，每百万token的价格可以达到45美元，甚至150美元。受限于GPU性能及高速推理芯片，中国开源模型仍然盘踞在高吞吐、低单位价值的那一个角落。
