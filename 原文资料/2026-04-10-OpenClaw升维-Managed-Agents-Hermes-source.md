- 2月底推出，GitHub已狂揽4万星

## 苦小龙虾久矣

### 安全问题
- ClawHub上高达36.8%的插件被查出存在严重漏洞或被投毒
- 「ClawHavoc」攻击事件：恶意插件伪装成办公助手，扫荡用户本地 `~/.openclaw` 目录
- 窃取飞书、邮箱聊天记录，电子钱包私钥和交易所密码

### 架构痛点
- 各种插件和密钥东拼西凑的"缝合怪"
- 狂烧Token、抽干钱包
- 一次版本更新直接白屏
- Anthropic收紧政策，限制通过套餐额度"绕道"支撑外挂Agent，拿OpenClaw开刀
- 必须走按用量计费的API

## Managed Agents架构革命

### 核心思路：大脑和双手彻底分开
OpenClaw的核心痛点是Harness和模型能力的深度耦合。当Claude从Sonnet进化到Opus，原本的Harness逻辑变成冗余代码垃圾。

Managed Agents围绕稳定接口构建，即使Harness逻辑变化，接口依然稳定。

### 三个标准组件（致敬Unix）

| 组件 | 功能 |
|------|------|
| **Session（会话层）** | "仅追加"的日志，记录发生的一切，独立于模型存在，重启不丢失 |
| **Harness（框架层）** | Agent的"循环逻辑"，负责调用Claude并分发指令到工具 |
| **Sandbox（沙箱层）** | 绝对隔离的计算环境，Claude在此运行代码、修改文件 |

**关键**: 大脑（模型）不需要知道手（沙箱）在哪里运行。

### 从"工具嫁接"到"AI原生系统"

#### 解耦革命：让容器变成"牲畜"
- 早期方案：会话+Harness+沙箱封装在单一容器 → "宠物效应"（娇贵不可替代）
- Managed Agents：容器变成"牲畜"，Harness本身也变成"牲畜"
- Harness不再驻留在容器内，像调用工具一样调用容器：`execute(name, input) → string`

#### 会话即日志：突破上下文窗口
- 会话不是窗口，而是日志
- 外部存储 + 按需interrogation + 框架转换 → 突破上下文限制

#### 规模化红利
- p50 TTFT下降约60%，p95下降90%以上
- Claude可同时对多个执行环境进行推理

#### 安全隔离
- Token保存在沙箱外部的安全保险库
- AI通过智能体发起调用，自己看不见密钥
- Prompt Injection攻击失去物理目标

## Hermes Agent：一边干活一边升级自己

### 三层学习闭环

**第一层：记忆**
- 两份长期记忆文件：`MEMORY.md` 和 `USER.md`
- 支持基于FTS5的跨会话检索 + 大模型摘要
- 不会像其他智能体那样开新会话就"失忆"

**第二层：技能**
- 完成复杂任务后，回顾关键步骤，拆出来，记踩的坑，写验证方法
- 整理成结构化技能文件
- 技能不是写死的，会继续改进
- 别的智能体在消耗上下文，Hermes在沉淀上下文

**第三层：训练数据**
- 内置批量轨迹生成能力
- 接入Atropos强化学习环境
- 任务轨迹可直接拿去训练下一代模型

### 核心意义
打通链路：一次任务 → 记忆 → 技能沉淀 → 轨迹回收 → 流回训练
