## 核心痛点

传统科研的困境：
- 一个人方向偏了没人提醒
- 遇到歧义没人讨论
- 结果不对只能反复试错
- "自动科研"只是封装成无人流水线

**真正高效的科研**：像实验室一样，不同角色同时推进，不同路径并行展开。

---

## 系统架构

### 金字塔式分层架构

```
研究方向设定
    ↓
方法设计与实验规划
    ↓
代码实现与结果分析
```

- 每一层由专属Agent负责
- 任务队列与上下文紧密连接
- 上层决策可根据下层结果动态调整

### 三种模式

| 模式 | 说明 |
|------|------|
| **Lab讨论模式** | 多方向并行调研，跨方向讨论达成共识，生成统一假设 |
| **Lab独立研究模式** | 多方向并行调研，各方向共享知识库独立生成假设 |
| **论文复现模式** | 单Agent全流程复现目标论文的方法与实验 |

## Claw Code Harness

**核心创新**：不只是"写代码"，而是"把研究想法落成可运行实验"。

### Turn Loop机制
```
读取代码库/数据集/模型检查点
    ↓
理解任务 → 编写main.py → 运行测试
    ↓
定位报错 → 继续修复
    ↓
循环直到成功
```

### Experiment Harness
- 时间预算控制
- 指标上报
- 异常值校验
- 生成标准化results.json

**目标**：AI生成的不是demo，而是真正能落地、能复现、能被继续优化的研究代码。

## 从"单一"到"群体"智能

### 具身智能实验室示例

**场景**：你是PI，三名研究员方向分别是VLM、VLA、World Model，研究video action model落地方向。

**讨论前分歧**：
- **World Model研究员**：可控性、安全性、在线重规划
- **VLA研究员**：train with video, infer with action
- **VLM研究员**：任务理解、执行监控、异常预警

**讨论后共识**：
- 训练阶段：视频监督学习动态表征
- 执行阶段：保留直接动作输出，低时延闭环控制
- 系统上层：planning/safety layer做重规划
- 执行旁路：步骤理解、异常监控、anticipation

**关键**：分歧被展开，假设被暴露，证据被对齐，路线被重组。

## 项目示例

### 1. Hallucination量化研究
- **目标**：系统化量化大模型幻觉
- **难点**：缺乏统一标准、错误有"表面合理性"、多步推理放大
- **方法**：结构化拆解输出流程，多维度一致性分析

### 2. PhyCustom复现
- **目标**：在FLUX模型上复现物理属性可控生成
- **难点**：物理属性难以准确表达、对细节高度敏感
- **方法**：嵌入完整实验链路，关键步骤约束与追踪
