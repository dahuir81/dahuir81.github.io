## 泄露内容
昨晚 X 上泄露了 Claude Code 即将上线的更新：截图验证、安全扫描、设计探索、暗黑模式、登录系统、跨多个代码仓库的统一工作界面。网友评价："这可能是 Lovable 的全栈竞争对手"。

## 核心观点：从卖水到修管道

Anthropic 的生意本质是卖 API（按 token 付费=算力批发），但 token 价格持续下跌，纯卖 API 是价格战。把算力包装成产品（Claude Code 订阅、Cowork 企业版、Managed Agents），利润率可高 10 倍。

| 模式 | 本质 | 利润率 | 迁移成本 |
|------|------|--------|----------|
| API | 卖水 | 低 | 低（几行代码切换） |
| 产品 | 修管道 | 高 | 高（工作流锁定） |

## 吃掉自己的客户

Claude API 的付费大客户（Cursor、Lovable、Bolt）正面临危机：
- **Cursor**：核心能力来自 Claude API，当 Claude Code 自己做同样的事，护城河极薄
- **Figma**：当 AI 模型可直接生成 UI 和代码，设计工具的价值被侵蚀

## 平台收割应用层的历史规律

| 平台 | 收割对象 | 方式 |
|------|----------|------|
| Apple | Spotify | Apple Music 预装+30%苹果税 |
| AWS | 第三方数据库/队列 | 自建同功能服务 |
| Microsoft | Slack | Teams 捆绑 Office |
| Anthropic | Cursor/Lovable | Claude Code 直接对标，模型能力不对称 |

**关键差异**：AWS 花几年才开始挤压生态，Anthropic 从 Claude Code 发布到全栈开发环境不到一年。

## 应用层还有活路吗？

**正面案例**：Adobe 从应用层往下吃基础设施，先有 Photoshop，再做 Creative Cloud，最后用 Firefly 嵌入 AI。但做模型门槛远高于做应用。

**可能的出路**：
1. 专业场景、垂直领域、企业定制（模型公司不想做的脏活累活）
2. 模型中立策略（不绑定任何一家模型）
3. 前提是模型差距缩小到可忽略

## 结论

模型公司做应用有天然优势（第一时间获得新能力），但如果模型差距缩小，应用层可用"模型中立"作为卖点。风水轮流转，去年 GPT 断层领先，今天 Claude 强势，未来谁领先还不确定。
