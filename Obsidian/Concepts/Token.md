# Token（AI时代的"度"与"流量"）

**定义**: AI模型处理文本的基本单位，既是计量单位，也是价值载体  
**相关概念**: [[Skill化]]、[[AI替代]]、[[知识蒸馏]]  
**流行语**: "散是Token，聚是skill"

---

## 双重含义

### 1. 技术定义
- **计量单位**: 模型处理文本的最小单元（单词片段或字符）
- **成本单位**: API按Token计费（输入+输出）
- **上下文窗口**: 模型一次能处理的Token数量限制

### 2. 隐喻意义（网友梗）
> "被毕业的同事其实并没有消失，他们只是被蒸馏成了Token，换成另一种形式陪伴你。"

**散是Token，聚是skill**:
- 人的经验、知识被拆解成Token
- Token被训练、聚合成Skill
- 人本身被替代

---

## Token经济

### 成本结构
| 项目 | 说明 |
|------|------|
| 输入Token | 用户发送的文本 |
| 输出Token | AI生成的回复 |
| 计费方式 | 通常 $X/1M tokens |

### 价值转换
```
人的经验 → Token化 → 训练 → Skill → 替代人
   ↑                                    ↓
   └──────── 价值被提取，人被淘汰 ────────┘
```

---

## 在Skill化中的角色

**蒸馏过程**:
1. 人的工作产出 → 文本/代码/决策记录
2. 文本 → Token序列
3. Token → 训练数据
4. 训练 → Skill（可执行的能力）
5. Skill → 替代原人

**关键洞察**:
> "你的专业知识、判断逻辑、处理边缘情况的直觉，这些构成你不可替代性的东西，恰恰是最容易被提取和蒸馏的东西。"
