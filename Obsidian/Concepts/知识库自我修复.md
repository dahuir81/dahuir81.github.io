# 知识库自我修复 (Self-Healing Knowledge Base)

**定义**: LLM自动扫描、识别问题并修复的知识库维护机制  
**来源**: Karpathy提出，陆徐洲实践验证  
**核心**: 不用盯着每个细节，LLM帮你查漏补缺

---

## 典型修复场景

### 1. 断链修复
- **扫描**: 检查所有[[双向链接]]
- **识别**: 找出没有对应文件的链接
- **修复**: 自动补建高频概念页

**陆徐洲案例**:
- 32个断链 → 自动补建7个 → 降至25个

### 2. 孤儿页面检测
- 没有入链的页面
- 自动提示需要建立关联

### 3. 概念缺失补全
- 高频提及但没有独立页面
- 自动提取并创建

---

## 实现方式

**定期执行**:
```
扫描链接 → 识别断链/孤儿页 → 自动修复/提示
```

**工具**: Claude Code / OpenClaw Agent

---

## 与人工维护对比

| 维度 | 人工维护 | 自我修复 |
|------|---------|---------|
| 频率 | 偶尔想起 | 定期自动 |
| 覆盖率 | 容易遗漏 | 全面扫描 |
| 及时性 | 滞后 | 即时 |
| 成本 | 高（需人工投入） | 低（LLM自动） |
