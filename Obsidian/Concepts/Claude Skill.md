# Claude Skill

**定义**: 一套指令，打包成文件夹，教Claude怎么处理特定任务  
**提出者**: Anthropic  
**核心思想**: 教一次，以后直接用

---

## 解决的问题

**痛点**: 每次对话都要重新解释偏好、流程、专业背景

**方案**: 把指令打包成Skill，Claude自动加载执行

---

## 核心设计理念

### 1. 渐进式披露

```
第一层: YAML前置信息 → 精简，每次加载
    ↓
第二层: SKILL.md正文 → 任务相关时加载
    ↓
第三层: 链接文件 → 按需查阅
```

**好处**: 省token，保深度

### 2. 可组合性
- 可同时加载多个Skill
- Skill间和平共处

### 3. 可移植性
- Claude.ai / Claude Code / API 通用
- 写一次，到处用

---

## 文件夹结构

```
your-skill/
├── SKILL.md      # 必须 - 主文件
├── scripts/      # 可选 - 可执行代码
├── references/   # 可选 - 参考文档
└── assets/       # 可选 - 模板资源
```

**命名**: kebab-case（短横线连接小写）

---

## YAML前置信息

```yaml
---
name: skill-name
description: 做什么 + 什么时候用
license: MIT          # 可选
compatibility: ...    # 可选
metadata:             # 可选
  author: ...
  version: 1.0.0
---
```

**安全限制**:
- 禁止 `<` `>`
- 不能以"claude"/"anthropic"开头

## Skill vs MCP

| 维度 | MCP | Skill |
|------|-----|-------|
| 层级 | 连接层 | 知识层 |
| 作用 | 连接服务 | 教怎么用 |
| 提供 | 能做什么 | 应该怎么做 |

**配合**: MCP提供能力，Skill固化流程

---

## 三类用例

1. **文档创建** - 前端设计、报告生成
2. **工作流自动化** - 多步骤流程
3. **MCP增强** - 为MCP加工作流程

---

## 五大设计模式

| 模式 | 场景 |
|------|------|
| **顺序编排** | 按顺序执行多步骤 |
| **多MCP协调** | 跨多个服务 |
| **迭代优化** | 多轮提升质量 |
| **情境感知** | 不同上下文选不同工具 |
| **领域智能** | 嵌入专业知识 |

---

## 测试与迭代

### 三类测试
- **触发测试** - 对时机加载
- **功能测试** - 输出正确
- **对比测试** - 比无Skill更好

### 迭代信号
- **触发不足** → description加关键词
- **触发过度** → 加负触发词
- **执行问题** → 改进指令
