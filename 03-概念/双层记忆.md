# 双层记忆

**定义**: [[Hermes Agent]] 采用的记忆架构，将持久记忆与历史会话检索分离，避免 prompt 被污染。

## 架构分层

### L1 持久记忆（Frozen Snapshot）
- **MEMORY.md**: Agent 自身的笔记
- **USER.md**: 用户画像
- 会话开始时作为 frozen snapshot 一次性注入 system prompt
- 整个会话期间不变，容量写死，超了由 Agent 自己压缩
- 更新只在会话结束后收拢

### L2 历史会话搜索
- 全量会话存入 SQLite
- 配合 FTS5 全文索引
- 需要回忆"上周是不是讨论过 XX"时才主动检索
- 避免把所有历史一股脑塞进 prompt

### L3 外挂 Memory Provider（可选）
- 如 Mem0 等跨会话用户建模
- 可与双层架构结合使用

## 设计价值
- 解决"边推理边写"导致 prompt 污染的问题
- 降低记忆维护成本，提升跨会话复用能力
