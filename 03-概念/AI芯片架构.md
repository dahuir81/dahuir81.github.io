# AI芯片架构

## 核心概念

AI芯片架构设计围绕三个核心矛盾展开：
1. **算力 vs 内存带宽** - 计算快但数据供不上（显存墙）
2. **通用性 vs 专用性** - GPU全能但效率低，ASIC高效但不灵活
3. **单芯片 vs 集群扩展** - 单卡再强也有限，集群通信开销大

## 关键架构组件

### 1. 计算单元

#### TensorCore / MXU
- **功能**: 矩阵乘法加速（深度学习核心操作）
- **设计要点**: 支持FP8/FP16/BF16等低精度计算
- **代表**: NVIDIA TensorCore, Google MXU, AMD Matrix Core

#### SparseCore
- **功能**: 稀疏计算加速（Embedding、推荐系统）
- **设计要点**: 专门处理稀疏矩阵，与Dense计算并行
- **代表**: Google TPU Ironwood第四代SparseCore

### 2. 内存子系统

#### HBM (High Bandwidth Memory)
- **代际**: HBM2e → HBM3 → HBM3e
- **关键指标**: 容量、带宽、能效
- **瓶颈**: 内存墙 - 算力增长远快于内存带宽

#### 共享内存架构
- **传统**: 每芯片独立显存，通过NVLink/网络通信
- **革命性**: 全局统一寻址（如Ironwood 1.77PB共享内存）
- **优势**: 消除通信开销，简化编程模型

### 3. 互联架构

#### 片内互联
- **片上网络(NoC)**: 芯片内部各模块通信
- **Die-to-Die**: 多Die芯片内部互联（如Ironwood双Die）

#### 片间互联
- **电互联**: NVLink, PCIe, UCIe
- **光互联**: OCS (Optical Circuit Switching)
  - 优势: 低延迟、高带宽、低功耗
  - 代表: Google OCS, Ayar Labs

## 系统级架构

### 单芯片 → 托盘(Tray) → 机架 → 集群

```
Ironwood架构示例:

单芯片: 4614 TFLOPS, 192GiB HBM3e
    ↓
Tray: 4芯片, ~20 PFLOPS, 液冷散热
    ↓
机架: 多个Tray, OCS互联
    ↓
SuperPod: 9216芯片, 42.5 Exaflops, 1.77PB共享内存
```

### 关键设计原则

1. **并行液冷**: 每芯片独立水路，避免串联温度不均
2. **全对全互联**: 任意两芯片间高带宽直连
3. **故障隔离**: 单点故障不影响整体，支持热插拔
4. **动态切片**: 根据任务需求灵活分配集群规模

## 能效优化

### 为什么能效比越来越重要？

- 数据中心电力成本占总成本30-40%
- 电力供应成为扩建瓶颈
- 碳中和压力

### 优化手段

| 手段 | 效果 | 代表 |
|------|------|------|
| 专用架构 | 去掉通用计算冗余 | TPU vs GPU |
| 低精度计算 | FP8 vs FP32，4倍能效 | NVIDIA Transformer Engine |
| 先进制程 | 3nm vs 5nm | TSMC N3E |
| 先进封装 | Chiplet, 3D堆叠 | CoWoS, EMIB |
| 液冷散热 | PUE从1.6降到1.1 | 谷歌第三代液冷 |

## 峰值算力对比（TPP）

### 什么是 TPP？

**TPP（Total Processing Performance，总处理性能）**并非一般性能指标，而是**美国商务部BIS在出口管制中使用的监管指标**（EAR 3A090）：

```
TPP = 2 × MacTOPS × 位宽
```

- 乘加运算峰值 × 2（乘+加各算一次）× 数据位宽（FP16=16, FP8=8）
- 用于判定芯片是否受出口管制（≥4,800 全面禁售，1,600–4,800 需许可证）
- Morgan Stanley采用TPP是为了**在美国统一监管口径下**对比中外芯片

Morgan Stanley在7nm同代工艺下的TPP对比：

| 芯片 | TPP | vs A100 |
|------|-----|---------|
| NVIDIA A100 | ~12,000 | 基准 |
| 华为昇腾 910C* | ~13,000 | **超越** |
| 寒武纪 MLU 800* | ~11,500 | 接近 |
| 华为昇腾 910B* | ~8,000 | 67% |
| 沐曦 MXN100* | ~8,000 | 67% |
| 华为昇腾 910B | ~6,000 | 50% |
| 燧原 CloudBlazer* | ~4,000 | 33% |
| 昆仑芯 XPU* | ~3,800 | 32% |

_* = Morgan Stanley估算值_

**核心含义**: 在美国自己的出口管制公式下，中国7nm芯片的TPP已触及A100水平——即中国芯片设计能力已达到"被禁售门槛"。但TPP是**理论值**，不代表实际性能。

## 行业趋势

### 1. 从通用到专用
- GPU仍是主流（CUDA生态护城河）
- 但TPU、Trainium、Gaudi等专用芯片崛起
- 推理场景专用芯片（如Groq）爆发

### 2. 从单卡到集群
- 大模型训练需要万卡级集群
- 集群架构设计成为核心竞争力
- 互联技术（光互联）成为关键

### 3. 从硬件到全栈
- 芯片、软件、算法、数据全链路优化
- 谷歌：TPU + JAX + Gemini
- NVIDIA：GPU + CUDA + Megatron
