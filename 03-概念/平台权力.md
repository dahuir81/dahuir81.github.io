# 平台权力

**类型**：商业概念 / 平台经济学
**来源**：Anthropic 2026 年对第三方框架的政策封锁

## 定义
平台方不仅在产品层竞争，还通过修改规则、限制访问权限来直接打击生态中的竞争对手。这种权力比产品能力更直接、更致命。

##  Anthropic 案例（2026）

### 产品层面
- Claude Code Channels：直接抄走 OpenClaw 的 IM 远程操控核心功能
- 功能描述中性："更好的工具、更低的门槛、更流畅的工作流"
- 但杀意藏在功能描述里

### 规则层面
- 2026年4月4日，以违反服务条款为由，**切断第三方 Agent 框架通过订阅凭证访问 Claude 的权限**
- OpenClaw 等框架用户必须改用按量计费 API，成本可能增加数十倍
- 直接在 OpenClaw 订阅服务上做限制，不让第三方用订阅服务

## 核心逻辑
**"在 Anthropic API 上构建商业模式，天花板从来不是技术，而是 Anthropic 的意愿"**

这与两年前 Sam Altman 的"套壳 OpenAI 的公司都会死"是同一逻辑。Anthropic 的 toB 生态更庞大，创业公司依赖更深。

## 与产品竞争的区别
| 维度 | 产品竞争 | 平台权力 |
|------|----------|----------|
| 方式 | 做更好的功能 | 修改规则 |
| 影响 | 对手可能输 | 对手失去存在前提 |
| 速度 | 渐进 | 瞬间 |
| 防御 | 可以做差异化 | 几乎无法防御 |
