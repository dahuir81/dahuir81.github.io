## 核心定义

长上下文（Long Context）指 AI 模型处理和利用超长输入序列的能力，上下文窗口从传统 4K~32K Token 扩展至百万 Token 级别。这是实现深度文档理解、全代码库分析、长视频理解等复杂任务的关键基础能力。代表产品包括 Kimi（200万字）、Claude（200K+ tokens）。

## 关键特征

- **百万 Token 窗口**：前沿模型已支持 200K~1M+ Token 的上下文窗口
- **信息检索精度**：长上下文下模型需要保持"大海捞针"级别的信息定位能力，避免中间位置遗忘
- **KV Cache 膨胀**：上下文越长，KV Cache 占用显存线性增长，对硬件提出严峻挑战
- **注意力计算复杂度**：标准 Attention 为 O(n²)，长文本导致计算量爆炸，需要优化算法
- **位置编码扩展**：RoPE 等位置编码方案需要外推至远超训练长度的范围
- **分层处理策略**：结合摘要、索引、检索等技术，在有限窗口内实现超长文档的有效处理
- **信息保真度**：长距离依赖容易丢失，需要优化注意力机制和位置编码
