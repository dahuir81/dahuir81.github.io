# 长期记忆系统 (Agent Memory)

**类型**: 概念（AI 架构/系统设计）
**关联文章**: [[2026-04-09-阿里云百炼记忆库-source]]
**创建/更新**: 2026-04-09

## 定义

为 AI Agent 提供跨会话的长期记忆能力，使 Agent 能够"越聊越懂用户"，实现个性化交互体验。

## 核心架构

```
对话 → 提取 → 存储 → 检索 → 注入 → 个性化回答
```

### 四大模块

| 模块 | 功能 | 说明 |
|------|------|------|
| 提取 | 自动识别关键信息 | 用户画像、偏好、家庭关系等 |
| 存储 | 结构化持久化 | 分类存储，支持多场景规则 |
| 检索 | 语义召回 | 基于语义理解匹配相关记忆 |
| 注入 | 上下文补充 | 将召回记忆附加到对话上下文 |

## 动态记忆治理

模拟人类记忆机制：
- **自动更新**: 跟踪信息变化，过滤记忆冲突
- **自主遗忘**: 对低频、过时信息进行衰减或归档
- **防止上下文污染**: 确保检索结果的高信噪比

## 行业方案对比

| 平台 | 方案 | 特点 |
|------|------|------|
| 阿里云百炼 | 记忆库系统 | 四大模块，OpenClaw 插件，限时免费 |
| Hermes Agent | SQLite + FTS5 全文检索 | MEMORY.md(~800 tokens) + USER.md(~500 tokens) + state.db |
| OpenClaw | memory/*.md + MEMORY.md | 本地文件存储，手动管理 |
| Honcho/Mem0/Hindsight | 第三方记忆插件 | 8+ 外部记忆插件生态 |

## 商业价值

- 教育辅导：记住学生进度和薄弱点
- 智能客服：记住用户历史问题和服务记录
- 情感陪伴：建立长期用户关系
- 智能硬件：个性化用户体验
