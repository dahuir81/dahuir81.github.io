# Context工程

**类型**: AI工程方法论

## 定义
Context 工程是将企业数据、权限、行动、记忆整合为结构化上下文，使大模型能够理解和操作企业业务的方法论。

## 公式
```
Context = 数据 + 权限 + 行动 + 记忆
```

## 四个维度

### 1. 数据（Data）
- 将 SQL 表转化为业务对象（如"飞机"、"航班"）
- 通过知识图谱关联多个对象
- 添加详细元数据描述

### 2. 权限（Permission）
- 继承源系统数据权限
- 确保合规访问
- 动态权限校验

### 3. 行动（Action）
- 通过 API 调用源系统能力
- 执行实际业务操作（如发优惠券、调整库存）
- 操作结果反馈

### 4. 记忆（Memory）
- 记录每次执行结果
- 持续优化策略
- 形成企业知识库

## 应用案例

### Palantir Foundry
- 将航班系统的 "Aircraft id" 字段转化为 "飞机" 对象
- 知识图谱关联：A零件 → 3家供应商（30%、30%、40%供货比例）
- AI 制定采购策略时，自动考虑历史执行结果

## 核心价值
- 大模型不需要理解复杂 SQL，面对的是直观业务实体
- 从 "懂全世界" 到 "懂我的公司"
