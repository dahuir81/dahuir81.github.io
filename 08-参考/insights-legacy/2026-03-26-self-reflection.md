## 🔧 系统性改进建议

### 1. 网络依赖方案的双轨设计
任何依赖外部网络的流程，必须准备：
- 主路径（国际网络）
- 备用路径（国内可访问）

### 2. 新环境检查清单
进入新系统前必须执行：
```bash
systemctl --user list-units | grep openclaw  # 查看自动服务
netstat -tlnp | grep 8080                   # 查看端口占用
```

### 3. 工具故障降级策略
| 工具 | 故障时降级方案 |
|------|---------------|
| web_search | browser直接抓取已知源 |
| browser | 使用预设/缓存内容 |
| feishu_im | 写入本地文件待同步 |

### 4. 流程设计原则
- 先画最简单流程图
- 问自己："能否再简化一步？"
- 验证最小可行后再扩展
